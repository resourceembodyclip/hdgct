# LinkVault Resource Aggregator

LinkVault is a production-grade technical resource aggregation and external link management system designed for developers, technical researchers, and content curators who need to organize, categorize, and distribute large volumes of external reference links. The project addresses the common challenge of managing unstructured link collections by providing a standardized indexing framework, metadata extraction pipelines, and automated health monitoring for external resources.

Target users include open-source maintainers building resource hubs, technical documentation teams managing reference libraries, and individual developers who maintain curated link collections for personal knowledge bases. LinkVault processes batches of URLs, extracts article metadata, generates searchable indices, and provides a lightweight web interface for browsing categorized resources. The system is built to handle batches of up to 200 links per processing run and supports incremental updates, making it suitable for ongoing content curation workflows.

## 功能概览

- **批次化链接处理引擎** - 支持按批次（当前第34/56批）导入和管理大量URL，自动去重并记录处理状态，每批次最多处理200个资源链接。

- **元数据自动提取器** - 从目标页面解析标题、发布时间、内容摘要和关键词，生成结构化索引数据，支持HTML和JSON两种输出格式。

- **资源健康状态监控** - 定期检测外部链接的可访问性，记录HTTP状态码和响应时间，自动标记失效链接并生成告警报告。

- **分类标签管理系统** - 支持为每个资源分配多个分类标签，内置技术文章、开发文档、视频教程、工具站点等预设分类，同时支持自定义标签。

- **全文检索与过滤器** - 基于标题和摘要的关键词搜索，配合标签、批次号、状态等多维度过滤器，快速定位目标资源。

- **数据导出与集成接口** - 提供RESTful API和批量导出功能，支持JSON、CSV和Markdown格式输出，便于与其他文档系统或静态站点生成器集成。

- **访问统计分析** - 记录资源被查看和点击的次数，生成简单的热度排行，辅助判断哪些链接对用户更有价值。

## 应用场景

- **技术文档库维护** - 技术文档团队可以使用LinkVault整理项目相关的参考链接、第三方依赖文档和社区讨论帖，确保文档中的外部引用可追溯、可验证。每批次导入后自动生成变更日志，便于版本管理。

- **开源项目资源页构建** - 开源项目维护者利用LinkVault批量整理社区提交的教程、博客和视频链接，自动生成资源列表页面，并通过健康监控功能及时发现失效链接，保持资源页的可用性。

- **个人知识管理（PKM）** - 开发者可以将浏览器收藏夹中的技术文章批量导出为文本列表，通过LinkVault导入后获得结构化索引，配合标签系统构建可检索的技术资料库，减少信息碎片化。

- **DevOps 监控集成** - 运维团队将LinkVault集成到监控告警链中，用于管理各类仪表盘链接、日志查询入口和内部工具地址，通过定时检测确保所有内部资源入口可用，当检测到失效链接时自动触发告警。

- **内容聚合站后端** - 内容站点运营者使用LinkVault作为外链管理后台，按批次组织编辑提交的参考来源链接，通过API将处理后的数据同步到前端展示页面，实现编辑与发布流程的分离。

## 快速开始

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装项目依赖（使用pipenv）
pip install -r requirements.txt

# 初始化数据库并运行开发服务器
python manage.py migrate
python manage.py runserver --batch 34 --total 180
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Python 3.9+ | 是 | 核心运行环境，推荐使用3.10或更高版本以获得更好的性能 |
| SQLite 3.35+ / PostgreSQL 12+ | 是 | 默认使用SQLite，生产环境推荐PostgreSQL存储元数据和索引 |
| requests 2.28+ | 是 | HTTP客户端，用于发起资源抓取请求和处理响应 |
| beautifulsoup4 4.11+ | 是 | HTML解析器，用于从目标页面提取元数据内容 |
| lxml 4.9+ | 是 | 高性能XML/HTML解析引擎，作为beautifulsoup的后端加速器 |
| schedule 1.1+ | 否 | 定时任务调度器，启用健康状态自动监控时必需 |
| redis 4.5+ | 否 | 缓存层，用于提升高频查询性能和存储临时处理状态 |
| pytest 7.2+ | 否 | 单元测试框架，仅开发和测试环境中使用 |
| docker 20.10+ | 否 | 容器化部署支持，通过Docker Compose可一键启动完整服务栈 |

## 项目结构

```
linkvault/
├── cmd/                                                 # 命令行入口目录
│   └── linkvault/                                       # 主CLI程序入口包
│       └── main.go                                      # 程序启动入口，解析命令行参数
├── internal/                                            # 内部核心包，不对外暴露
│   ├── batch/                                           # 批次处理引擎模块
│   │   ├── processor.go                                 # 批次处理器，协调抓取与索引流程
│   │   └── queue.go                                     # 批次任务队列管理，支持优先级排队
│   ├── crawler/                                         # 资源抓取与解析模块
│   │   ├── fetcher.go                                   # HTTP请求发起与响应处理
│   │   ├── parser.go                                    # HTML元数据解析器，基于goquery
│   │   └── middleware.go                                # 请求中间件，处理重试与限流
│   ├── storage/                                         # 数据持久化层
│   │   ├── sqlite.go                                    # SQLite驱动实现，用于开发环境
│   │   ├── postgres.go                                  # PostgreSQL驱动实现，用于生产环境
│   │   └── cache.go                                     # Redis缓存操作封装
│   ├── health/                                          # 健康监控模块
│   │   ├── checker.go                                   # 链接可用性检测器，定期巡检
│   │   └── reporter.go                                  # 状态报告生成器，输出监控结果
│   └── api/                                             # RESTful API服务模块
│       ├── handlers.go                                  # HTTP路由处理器，实现CRUD接口
│       └── middleware.go                                # 认证与日志中间件
├── pkg/                                                 # 可复用的公共库
│   ├── config/                                          # 配置加载与管理，支持YAML/ENV
│   └── utils/                                           # 工具函数集，包含字符串、时间等辅助
├── web/                                                 # 前端静态资源与模板
│   ├── templates/                                       # HTML模板文件，用于管理界面渲染
│   └── static/                                          # CSS/JS静态资源，提供基础UI样式
├── scripts/                                             # 运维与部署辅助脚本
│   ├── migrate.sh                                       # 数据库迁移脚本，自动执行schema变更
│   └── backup.sh                                        # 数据备份脚本，支持定时归档
```

## 常见问题

**Q: 批量导入链接时遇到HTTP超时或连接拒绝错误如何处理？**

A: LinkVault内置了指数退避重试机制，默认最多重试3次。如果仍然失败，系统会自动将链接标记为"抓取失败"状态并记录错误日志。您可以通过管理界面手动重新触发单个链接的抓取，或调整config.yaml中的timeout和retry参数来适应不同的网络环境。建议对于访问速度较慢的站点适当增加timeout值（例如从30秒调整到60秒）。

**Q: 数据库从SQLite迁移到PostgreSQL需要注意哪些事项？**

A: LinkVault提供了内置的迁移脚本（scripts/migrate.sh）。迁移前请备份SQLite数据文件，然后在配置文件中将storage.driver切换为postgres并填写正确的连接字符串（DSN）。运行迁移脚本时会自动执行schema转换和数据类型映射，其中SQLite的TEXT类型会被映射为PostgreSQL的TEXT，布尔类型映射为BOOLEAN。迁移完成后建议运行一次完整的链接健康检查，确保所有数据记录正确转换。若数据量超过10万条记录，迁移过程可能需要数分钟，建议在低峰期执行。

**Q: 如何为LinkVault添加自定义的元数据解析规则？**

A: 系统支持通过配置文件扩展自定义解析器。您可以在configs/目录下创建parser_rules.yaml文件，为特定域名或路径前缀指定CSS选择器规则（例如标题使用h1.article-title，发布时间使用time.pub-date）。系统内置了通用解析器作为fallback，当自定义规则不匹配时自动回退。如果目标站点结构复杂，您也可以编写Go代码实现parser.Parser接口，在internal/crawler/parser.go中注册新的解析器类型。
