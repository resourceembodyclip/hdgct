# LinkVault Resource Aggregator

LinkVault 是一个轻量级的技术资源与文章外链聚合平台，专注于对移动端技术文档、开发教程及行业资讯进行结构化归集与检索。项目面向开发者、技术研究者及内容策展人，帮助用户在碎片化的网络文章中快速定位高价值信息，降低信息过载带来的筛选成本。

项目以 URL 采集与元数据提取为核心，提供标签分类、全文检索及相关性排序功能，可作为个人知识库的后端数据源，也可嵌入团队文档系统作为外部链接中台。LinkVault 不存储文章本体，仅保留链接索引与摘要快照，确保合规性的同时维持极低的运维开销。

## 功能概览

**批量链接导入** 支持从文本文件、CSV 或 RSS 订阅源批量导入文章 URL，自动去重并校验可访问性。

**元数据自动提取** 对每条链接执行标题、发布时间、正文摘要及关键词的自动抓取，支持自定义 XPath 与正则解析规则。

**多级标签体系** 允许用户创建无限层级的分类标签，支持标签继承与别名映射，便于多维度组织资源。

**全文检索引擎** 基于倒排索引实现标题与摘要的快速搜索，支持模糊匹配、布尔表达式及日期范围筛选。

**可访问性监控** 定时检测已收录链接的有效性，自动标记失效链接并生成报表，支持死链批量清理。

**RESTful API 接口** 提供标准 JSON API 用于资源的增删改查、标签管理及搜索调用，便于与其他系统集成。

**数据导出与备份** 支持将资源列表导出为 JSON、CSV 或 Markdown 格式，数据库每日自动备份。

## 应用场景

个人技术博客的知识库建设。独立博客作者可以使用 LinkVault 收集行业优质文章作为写作参考，通过标签分类整理不同技术栈的资料，在撰写新内容时快速检索相关论点与案例。

技术团队的文档中心扩展。研发团队可将 LinkVault 集成至内部 Wiki 或 Confluence，用于统一管理团队收藏的外部技术文档、故障排查记录及架构设计参考，避免知识散落在个人浏览器书签中。

开源项目的资源导航页。开源项目维护者可以使用 LinkVault 生成项目相关的教程、插件生态及社区案例的链接列表，作为 README 或项目官网的延伸，为用户提供一站式外部资源入口。

技术培训的课程素材管理。培训机构或企业大学可以使用 LinkVault 按课程单元整理推荐阅读材料、视频链接及实验环境文档，学员可通过搜索快速获取课外扩展资源。

## 快速开始

以下步骤帮助你在本地环境快速启动 LinkVault 服务。

```bash
# 克隆代码仓库
git clone https://github.com/linkvault/linkvault-core.git
cd linkvault-core

# 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并创建管理员账户
python manage.py migrate
python manage.py createsuperuser

# 启动开发服务器
python manage.py runserver 0.0.0.0:8000
```

访问 http://localhost:8000 进入管理后台开始导入链接。生产环境部署请参考 `deploy` 目录下的 Docker Compose 配置文件。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 及以上 | 核心运行环境，低于此版本可能导致异步特性异常 |
| PostgreSQL | 14.0 及以上 | 主数据库，用于存储资源索引与标签体系 |
| Redis | 6.2 及以上 | 缓存与任务队列后端，用于定时监控与搜索缓存 |
| Elasticsearch | 7.17 或 8.x | 全文检索引擎，可选但推荐用于大规模数据检索 |
| Nginx | 1.20 及以上 | 生产环境反向代理与静态资源服务（可选） |
| Node.js | 18.x（仅前端构建需要） | 用于编译静态资源，开发模式可使用 CDN 替代 |
| Docker | 20.10 及以上 | 容器化部署方式所依赖的运行时 |

## 常见问题

**Q: 导入大量 URL 时出现超时或内存溢出如何处理？**

A: 建议使用命令行批量导入脚本 `scripts/import_batch.py` 并配合 `--chunk-size` 参数分批处理，每批建议 50-100 条。同时检查 Redis 队列是否正常运行，异步任务会将导入操作交由 Celery worker 执行，避免阻塞主进程。

**Q: 部分文章无法正确提取标题或摘要，如何自定义解析规则？**

A: 在管理后台的「解析规则」页面可以添加针对特定域名或 URL 前缀的自定义 XPath 与正则表达式。对于复杂的单页需求，可在 `src/apps/resources/parsers/` 下新建继承自 `BaseParser` 的专用解析器类，并在配置中注册。

**Q: 搜索返回结果与预期不符，相关性排序逻辑是什么？**

A: 默认排序基于 BM25 算法结合时间衰减因子，新发布的文章权重略高。若需要精确匹配，请在搜索关键词外使用双引号包裹短语。可在 `src/apps/resources/indexers/elastic.py` 中调整 `boost` 参数以改变标题与摘要的权重比值。
