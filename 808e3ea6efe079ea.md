# Vokav Mobile Article Indexer

Vokav Mobile Article Indexer 是一个面向移动端技术内容聚合与索引的开源工具集，专注于对 vokav.cn 域名下海量移动端技术文章进行结构化采集、元数据抽取与全文检索支持。该项目主要服务于移动开发技术文档维护者、个人知识库构建者以及需要批量处理移动端技术资讯的自动化流程开发者。

项目定位为轻量级技术资源外链汇总与索引中间件，不依赖复杂后端框架，通过静态解析与规则化提取，将分散的移动端文章链接转化为可机读的索引数据集，便于集成至静态站点生成器、搜索引擎或自定义推荐系统。目标用户包括移动端工程师、技术内容运营人员以及开源技术文档维护者。

## 功能概览

**批量链接规范化** 对输入的文章链接列表进行去重、格式校验与域名一致性检查，确保所有资源指向 mobile.vokav.cn 或 www.mobile.vokav.cn 的有效子路径。

**元数据启发式抽取** 从文章详情页的 HTML 结构中自动推断标题、发布时间、正文摘要及分类标签，无需依赖外部 API 或数据库。

**索引快照生成** 定期生成全量资源索引的 JSON 和 CSV 快照，支持增量更新与版本追溯，便于本地存档或云端同步。

**搜索词高亮匹配** 内置简单的关键词匹配引擎，可在索引结果中快速定位包含特定技术术语（如 React Native、Flutter、Kotlin）的文章条目。

**多格式导出适配** 支持将索引结果导出为 Markdown 表格、HTML 无序列表或纯文本清单，直接适配技术博客、项目文档或邮件周报场景。

**定时任务钩子** 提供 Cron 表达式配置接口，可挂载至 GitHub Actions 或 Jenkins 实现每日自动拉取与索引更新。

## 应用场景

**移动开发周报自动生成** 技术团队负责人可配置定时任务，每周自动拉取 vokav.cn 上新增的移动端文章，生成带摘要的周报 Markdown 文件，直接提交至团队知识库仓库。

**个人技术阅读清单管理** 开发者可将感兴趣的移动端文章链接统一提交至项目索引器，项目自动为其生成阅读进度跟踪表，并支持按标签（性能优化、UI 渲染、网络协议等）分组展示。

**技术文档站外链校验** 静态站点生成器（如 VuePress、Docusaurus）的维护者可利用项目提供的链接检查模块，批量验证文档中引用的 vokav.cn 外链是否仍可访问，并自动更新失效链接。

## 快速开始

以下步骤适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git 与 Node.js 18.x 或以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/vokav-mobile-indexer.git
cd vokav-mobile-indexer

# 安装依赖
npm install

# 运行索引器（默认处理项目内置的示例链接列表）
npm run index
```

## 常见问题

**Q: 索引器能否处理 m.mobile.vokav.cn 与 www.mobile.vokav.cn 混合输入的链接列表？**

A: 可以。validator 模块会自动识别子域名差异，并将所有链接统一规整为标准化格式后存入索引。两条子域名指向的服务器可能采用不同的缓存策略，但项目已实现分别采集与合并去重逻辑，最终索引数据集不会出现重复条目。

**Q: 如果目标文章页面发生结构改版，导致元数据抽取失败，项目会如何处理？**

A: 项目内置了降级策略。当抽取规则匹配失败时，extractor 会退回至全文搜索关键词（如 title、pub-date）的二次匹配模式，同时将该条记录标记为 partial 状态并记录解析错误日志。项目维护者会定期收集日志中的解析失败样本，并在后续版本中更新抽取规则配置文件。用户亦可自行修改 config/custom.json 中的 selector 映射以适配新结构。

**Q: 索引数据是否可以增量更新，而不是每次全量重建？**

A: 项目从 1.2.0 版本起支持增量索引模式。通过在配置文件中设置 incremental 为 true，系统会比对当前链接列表与最近一次快照的差异，仅对新链接或上次拉取失败的链接发起请求。已完成索引且状态码为 200 的链接将被跳过，大幅缩短批量处理时间。
