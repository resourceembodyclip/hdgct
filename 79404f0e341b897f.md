# LinkVault - 技术资源外链元聚合系统

LinkVault 是一个面向技术内容创作者、开发者文档维护者及开源社区运营者的外链资源批量管理与结构化归档平台。本项目并非传统意义上的爬虫或采集系统，而是一套围绕特定领域知识节点进行外链元数据整理、分类索引与状态监控的静态资源聚合框架。

项目核心定位在于解决技术团队在编写文档、搭建知识库或运营技术社区时面临的“链接散落、失效不可知、引用无回溯”三大痛点。LinkVault 不对原始内容进行转载或存储，而是通过标准化的外链清单（Manifest）机制，将大量分散的参考链接按业务主题、内容类型与时效性等级进行归并，并生成人类可读与机器可解析的双模输出。目标用户包括技术博客作者、API 文档编写者、技术 Newsletter 编辑以及企业内部知识管理专员。

## 功能概览

**批量外链导入与校验** 支持从 CSV、JSON 及纯文本列表批量导入原始 URL，自动执行协议一致性检查与域名可达性预检，并生成导入报告。

**多维度分类标签系统** 允许为每条外链标记主题分类、内容形态（文章/视频/工具/规范）、目标读者层级（入门/进阶/专家）及预期更新频率，便于后续按场景筛选。

**链接状态周期性探测** 内置基于 HTTP 状态码与响应时间阈值的健康检查引擎，可配置定时任务对已收录链接进行存活检测，并标记失效、重定向或响应过慢的链接。

**结构化目录树生成** 根据用户设定的分类规则，自动将外链资源映射为文件系统风格的目录树结构，并生成对应的 Markdown 索引页，方便静态站点生成器直接引用。

**外链引用关系追溯** 记录每条外链被哪些本地文档或项目模块所引用，支持反向查询与影响面分析，在链接失效时快速定位受影响内容。

**导出与集成适配** 提供纯 Markdown 列表、JSON Schema、RSS Feed 以及 HTML 小部件等多种输出格式，可无缝嵌入 MkDocs、VuePress、Hugo 等主流文档框架。

## 应用场景

技术文档站点的参考链接规范化管理。当技术文档中包含大量“延伸阅读”或“参考资料”节时，维护者通常面临链接格式不统一、重复收录、失效后难以同步更新等问题。LinkVault 可通过统一的外链清单文件集中管理所有引用链接，并在文档构建阶段自动注入经过校验的最新链接列表，确保每个版本发布的文档均包含有效的外部引用。

技术类 Newsletter 或周报的素材积累。编辑人员在策划技术周报时，常需从大量来源中筛选优质内容并附上原文链接。LinkVault 允许编辑团队按主题标签（如 Rust 生态、前端工程化、数据库内核）预先分类存储候选链接，并结合状态探测功能筛除已失效或内容变更的页面，显著提升内容策展效率。

企业内部知识库的外部资源合规审计。企业知识库中涉及的外部链接可能包含商业竞品信息、第三方服务条款变更或开源协议更新内容。LinkVault 的链接状态探测与变更日志功能可帮助合规部门定期导出外部资源引用报告，及时识别需要更新或移除的外部依赖。

开源项目 README 与贡献指南的外链维护。开源项目通常在 README 中附带大量的社区资源、学习路径或插件索引链接。项目维护者可使用 LinkVault 维护独立的链接清单文件，并通过 CI 流水线自动检查清单中各链接的有效性，避免用户因访问失效链接而产生负面体验。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git 与 Node.js 18.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git

# 进入项目目录
cd linkvault

# 安装项目依赖
npm install

# 复制示例配置文件并编辑
cp .env.example .env

# 运行初始化构建，生成默认分类索引
npm run build

# 启动本地开发预览服务器
npm run dev
```

首次运行后，项目将自动生成 `data/manifest.json` 与 `docs/index.md` 两个核心文件。用户可通过编辑 `data/sources/*.txt` 目录下的文本文件来添加自定义外链列表，每次保存后开发服务器将自动重新构建索引。

## 项目结构

```
linkvault/
├── bin/
│   └── linkvault-cli.js          # 命令行入口，暴露 build/check/export 子命令
├── config/
│   ├── default.yaml              # 默认分类规则与探测参数
│   └── custom.schema.json        # 用户自定义分类的 JSON Schema 定义
├── src/
│   ├── core/
│   │   ├── manifest.js           # 外链清单的加载、合并与去重逻辑
│   │   ├── classifier.js         # 基于规则与关键词的自动分类引擎
│   │   └── resolver.js           # 处理相对路径、协议补全与 URN 标准化
│   ├── health/
│   │   ├── checker.js            # 基于 curl 的异步 HTTP 状态探测
│   │   ├── scheduler.js          # 定时任务调度与并发控制
│   │   └── cache.js              # sqlite3 缓存读写与历史趋势计算
│   ├── output/
│   │   ├── markdown.js           # 生成 Markdown 列表与索引页
│   │   ├── json.js               # 输出符合 LinkVault Schema 的 JSON
│   │   └── rss.js                # 生成最近变更链接的 RSS Feed
│   └── watcher/
│       └── file-watcher.js       # 监听源文件目录变更并触发增量构建
├── data/
│   ├── sources/                  # 用户放置原始外链列表的目录（*.txt）
│   ├── cache/                    # sqlite3 数据库文件存放位置
│   └── manifest.json             # 构建后生成的完整清单主文件
├── docs/
```

## 常见问题

**Q: LinkVault 是否会存储或缓存外部链接的完整网页内容？**

A: 不会。LinkVault 仅存储用户主动录入的链接元数据（URL、标题、分类标签、录入时间）以及每次健康检查返回的 HTTP 状态码、响应时间和重定向链信息。系统不抓取、不解析、不存储任何外部页面的正文或 HTML 结构，所有操作均符合 robots.txt 的通用访问规范。

**Q: 如果收录的链接数量很大（超过 1 万条），系统性能如何？**

A: LinkVault 的核心数据流设计为流式处理，清单加载采用基于 Node.js Stream 的逐行读取，避免将全部数据一次性载入内存。健康检查模块默认使用 16 个并发请求，并可通过 `config/default.yaml` 中的 `check.concurrency` 参数调整。对于超过 5 万条记录的部署场景，建议使用 PostgreSQL 替代默认的 sqlite3 缓存后端，相关配置已在 `config/custom.schema.json` 中预留。

**Q: 如何自定义输出 Markdown 的模板样式？**

A: 项目在 `src/output/markdown.js` 中使用了基于 Handlebars 的模板引擎。用户可在 `config/default.yaml` 的 `output.markdown.templateDir` 字段指定自定义模板文件夹路径，覆盖内置的列表渲染、分类标题以及页眉页脚区块。模板变量列表详见 `docs/output-formats.md` 的“模板变量参考”附表。
