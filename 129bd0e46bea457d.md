# Vokav Resource Aggregator

Vokav Resource Aggregator 是一个面向移动端开发者和技术研究人员的结构化外链资源汇总系统。该项目通过对 vokav.cn 移动站点下的技术文章进行系统性归类和索引，构建了一个涵盖前端开发、后端架构、移动端适配、性能优化及安全审计等多个技术领域的知识导航体系。项目定位为技术资源的中转站与发现平台，帮助开发者快速定位到特定技术问题的解决方案，减少在碎片化信息中检索的时间成本。

本项目不对原始内容做二次加工或转载，仅提供结构化的访问入口与分类索引服务。目标用户包括全栈工程师、移动端开发人员、技术架构师以及运维工程师。通过统一的入口，用户可以按技术领域、问题类型或文章编号进行快速检索，从而在 vokav.cn 已有的大量技术文档中高效定位所需信息。

## 功能概览

**结构化资源索引**：按照技术领域和文章类型对超过 180 个外链资源进行逻辑分组，提供清晰的分类导航。

**多维度检索支持**：支持按文章 ID、技术标签、问题场景等多种方式定位目标资源。

**移动端优先的访问优化**：所有资源链接均指向 wap.mobile.vokav.cn 子域，确保在移动设备上的访问体验。

**实时资源状态检测**：内置链接可用性检测模块，定期对收录的 URL 进行存活检查并标记异常状态。

**轻量级部署架构**：基于静态站点生成技术，无需数据库支持，可部署在任何支持 HTTP 服务的环境中。

**开放的数据格式**：资源索引数据以 JSON 和 YAML 双格式提供，方便第三方工具进行二次开发与集成。

**版本化变更追踪**：每次资源列表更新均记录变更日志，支持回溯历史版本的资源收录状态。

**自定义分类标签**：允许用户为特定资源添加本地标签，实现个性化资源管理。

## 应用场景

移动端开发者在遇到特定 API 使用问题或兼容性故障时，可通过本项目的分类索引快速定位到 vokav.cn 上对应的技术文章，获取解决方案或最佳实践参考。

技术团队在开展代码审查或架构评审前，可利用本项目的资源导航体系系统性地查阅相关领域的技术文档，确保评审依据的全面性和准确性。

运维工程师在处理线上故障时，可通过本项目按问题类型（如性能瓶颈、安全漏洞、配置错误）筛选相关资源，快速找到故障排查指引。

技术培训讲师可将本项目作为配套教学资源，按课程章节组织学员阅读对应的技术文章，实现理论与实践的有机结合。

## 快速开始

以下命令演示了如何获取项目源码、安装依赖并启动本地服务。

```bash
# 克隆项目仓库
git clone https://github.com/vokav/resource-aggregator.git

# 进入项目目录
cd resource-aggregator

# 安装依赖（使用 npm 或 yarn）
npm install

# 启动开发服务器
npm run dev

# 生产环境构建
npm run build

# 启动生产环境服务
npm start
```

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并在本地克隆您 Fork 后的副本。请确保在提交变更前将主仓库的更新同步到您的分支中，以避免合并冲突。

2. 在 `data/resources.json` 或 `data/resources.yaml` 中添加新的资源条目时，请确保 URL 完全按照原始格式录入，不做任何协议、域名或路径的修改。新增条目需包含 `id`、`url`、`category` 和 `description` 字段。

3. 修改分类标签体系时，请同步更新 `docs/classification.md` 文档中的分类定义，并确保所有受影响的资源条目已重新标注。提交前运行 `npm run validate` 命令进行数据格式校验。

4. 提交代码时请遵循 Conventional Commits 规范，使用 `feat:`、`fix:`、`docs:`、`chore:` 等前缀区分提交类型。提交信息应简明扼要地说明变更内容和动机。

5. 发起 Pull Request 之前，请确保所有测试用例通过（`npm test`），并且代码风格符合项目 ESLint 配置要求（`npm run lint`）。PR 描述中应清晰说明变更目的、影响范围以及测试覆盖情况。

## 常见问题

**Q：项目收录的资源链接访问时返回 404 错误，应该如何处理？**

A：如果发现某个资源链接无法访问，请先在 vokav.cn 主站上确认该文章是否已被迁移或删除。若确认资源已失效，可在 GitHub Issues 中提交链接失效报告，项目维护团队会定期审核并更新资源列表。您也可以使用 `scripts/health-check.sh` 脚本在本地进行批量链接检测。

**Q：如何批量导入新的资源链接？**

A：项目提供了 `scripts/import-resources.py` Python 脚本，支持从 CSV 或纯文本文件中批量导入资源链接。导入前请确保数据格式符合 `docs/classification.md` 中定义的分类规范。导入后运行 `npm run build` 重新生成索引文件。

**Q：项目是否支持自定义分类标签？**

A：支持。您可以在本地通过修改 `data/resources.json` 中每个资源的 `tags` 数组添加自定义标签。但请注意，自定义标签仅在本地有效，若希望将标签合并到主仓库中，需按照贡献指南提交 Pull Request，并更新分类规范文档。
