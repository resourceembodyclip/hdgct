# LinkVault Core

LinkVault Core 是一个面向技术团队与独立开发者的结构化外链与资源归档系统。该项目并非传统意义上的导航站或书签管理器，而是一个围绕高质量技术文章、工具链文档与社区讨论进行持久化引用与分类索引的轻量级数据层。其核心定位在于帮助用户从分散的收藏行为中建立可检索、可共享、可版本控制的外部知识引用库，特别适用于需要频繁查阅碎片化技术资料、维护项目外部依赖文档或构建团队共识知识图谱的场景。

LinkVault Core 通过解析用户提供的原始 URL 清单，自动生成带分类标签与元数据的资源索引表，并支持静态站点生成、Markdown 导出与 RESTful API 查询三种消费模式。项目本身不存储任何第三方内容副本，仅保留 URL 规范形式、获取时间戳与自定义标签，从而在遵守版权规范的前提下提供高效的信息路由能力。当前批次为第 27/56 批，共计收纳 180 个经过人工核验的技术资源链接，覆盖移动开发、后端架构、运维监控与学术文献等多个子域。

## 功能概览

**批量 URL 注入与规范化清洗**：接受纯文本或 CSV 格式的原始 URL 列表，自动去除多余空白字符、统一协议大小写并校验路径合法性，输出严格符合 RFC 3986 标准的链接集合。

**多维度标签分类引擎**：基于 URL 路径特征、域名指纹库与可自定义的正则规则集，为每个链接自动生成最多五个分类标签，例如 mobile-article、backend-reference、devops-toolchain 等。

**资源状态健康检查**：周期性对已收录链接发起 HEAD 请求，检测 HTTP 状态码变化，标记失效链接并生成变更报告，支持手动重试与忽略规则配置。

**静态站点生成器适配层**：内置 Hugo 与 VuePress 主题模板映射规则，可将索引数据直接输出为符合上述静态站点生成器目录结构的 Markdown 文件与 YAML 元数据头。

**全文检索与过滤查询**：基于 SQLite FTS5 扩展提供中英文混合全文搜索，支持按标签、域名、收录批次、状态码等字段进行组合过滤，查询结果以 JSON 或表格形式返回。

**增量更新与变更审计日志**：每次新增或删除链接均记录操作类型、时间戳与操作者 IP（可选），所有变更写入 append-only 日志文件，便于回溯与版本比对。

**批处理任务队列管理**：针对超过 100 条链接的批量导入操作，自动拆分为子任务并支持断点续传，任务状态通过本地 SQLite 表持久化，重启后不丢失进度。

## 应用场景

技术团队内部知识库构建：团队 Leader 可将日常阅读中发现的高价值技术博客、官方文档更新公告与社区最佳实践案例统一收录至 LinkVault Core，通过标签体系按项目或技术栈分类，新成员入职时可直接拉取全量索引快速了解团队关注的技术域。

开源项目外部依赖文档归档：开源项目维护者常需引用大量第三方库文档、API 参考链接与兼容性列表。LinkVault Core 可为每个开源项目维护独立的资源清单，并在版本发布前自动校验所有链接的有效性，避免发布后出现死链。

个人技术学习路线管理：开发者可使用 LinkVault Core 按照学习阶段（入门、进阶、高级）或技术主题（并发编程、网络协议、数据库调优）整理碎片化阅读材料，配合健康检查功能定期清理失效链接，保持学习资源池的新鲜度。

离线文档中心数据准备：在网络受限的内网开发环境中，运维人员可借助 LinkVault Core 导出所有链接的元数据列表，结合自动化下载脚本批量获取静态副本，构建内部镜像站点的初始导航文件。

## 快速开始

以下命令适用于 Linux/macOS 系统，需提前安装 Git 与 Node.js 18.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/core.git linkvault-core
cd linkvault-core

# 安装依赖
npm install

# 运行初始化向导，创建本地 SQLite 数据库与配置文件
npm run init

# 导入示例资源清单（包含当前批次 180 个链接）
npm run import -- --batch ./samples/batch-27.csv

# 启动本地 Web 查询界面
npm run dev
```

## 项目结构

```
linkvault-core/
├── bin/                                 # 可执行入口脚本
│   ├── cli.js                           # 命令行交互主程序，注册所有子命令

├── config/                              # 配置文件目录
│   ├── default.yaml                     # 默认配置项，包含数据库路径、超时阈值等
│   └── tags.rules.json                  # 标签生成正则规则集，用户可自定义扩展
├── src/                                 # 核心源代码目录
│   ├── adapters/                        # 输出适配器模块
│   │   ├── hugo.js                      # Hugo 静态站点格式输出
│   │   └── vuepress.js                  # VuePress 格式输出
│   ├── core/                            # 核心业务逻辑
│   │   ├── importer.js                  # URL 导入与去重逻辑
│   │   ├── health.js                    # 链接健康检查调度器
│   │   └── tagger.js                    # 标签分类引擎实现
│   ├── db/                              # 数据库访问层
│   │   ├── schema.sql                   # SQLite 表结构定义
│   │   └── repository.js                # CRUD 操作封装
│   └── utils/                           # 通用工具函数
│       ├── url-validator.js             # URL 规范化与校验
│       └── logger.js                    # 日志记录器，支持按级别输出
├── tests/                               # 单元测试与集成测试
│   ├── unit/                            # 各模块独立测试用例
│   └── fixtures/                        # 测试用模拟数据与样本 CSV

│   ├── user-guide/                      # 用户指南
│   ├── ops/                             # 运维手册
│   └── developer/                       # 开发者文档
```

2. 在 GitHub Issues 中搜索是否存在相关议题，若无则新建议题并详细描述所修复的问题或新增的功能，获得维护者反馈后再行开发。

3. 派生项目仓库至个人账号，创建以 feature/ 或 fix/ 为前缀的分支，遵循现有的代码风格与 ESLint 规则进行开发，并补充对应的单元测试用例。

4. 提交 Pull Request 前运行完整的测试套件与构建流程，确保所有已有测试通过，新功能的测试覆盖率达到 80% 以上。

5. PR 描述中需引用关联议题编号，列出变更摘要与测试结果，维护者将在 7 个工作日内进行 Review 并给出合并意见。

## 常见问题

**Q：导入 CSV 文件时提示编码错误，如何处理？**

A：LinkVault Core 默认要求 CSV 文件采用 UTF-8 编码，且首行为列头。若文件来源为 Windows 系统，建议使用 Notepad++ 或 VS Code 将编码转换为 UTF-8 without BOM。同时确认列分隔符为逗号或制表符，可在 config/default.yaml 中调整 delimiter 配置项。

**Q：健康检查模块对大量链接同时检测时是否会触发目标站点的反爬策略？**

A：健康检查模块默认启用并发限制，单次最多同时发起 5 个 HEAD 请求，且请求间隔不低于 200 毫秒。对于已知严格限制访问频率的域名，可在 tags.rules.json 中为该域名配置 rate-limit 覆盖参数，将间隔延长至 1 秒以上。此外，所有请求均携带标准 User-Agent 头并遵守 robots.txt 的 Allow 与 Disallow 规则。

**Q：能否在无网络环境下使用 LinkVault Core？**

A：可以。除健康检查与首次安装依赖外，LinkVault Core 的核心导入、查询与导出功能完全离线可用。用户可在有网络时预先安装所有 npm 依赖，并将 SQLite 数据库文件通过移动介质迁移至内网环境，所有操作均不依赖外部 API 服务。
